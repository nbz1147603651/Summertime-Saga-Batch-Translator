# 🔍 编译版本问题说明

## 问题描述

你的 STS 翻译工具在**扫描阶段卡住**了，原因是：

**游戏发布版本包含的是编译文件（.rpyc），而非源代码（.rpy）**

- ✗ `.rpyc` = 编译后的二进制文件，**无法直接编辑**
- ✓ `.rpy` = 源代码文本文件，**这是我们需要的**

---

## 🎯 解决方案

### 方案 A：使用源代码版本（推荐）

如果你有权访问游戏源代码或开发版本：

1. **获取源代码版本**
   - 从官方开发者处获取
   - 或从 GitHub / GitLab 等源代码仓库克隆
   - 确保包含 `src.rpa` 或已解压的 `scripts/` 目录中有 `.rpy` 文件

2. **重新解压**
   - 使用"解压存档"页面解压源代码版本的 `src.rpa`
   - 选择到 `.rpy` 文件所在的目录

3. **继续翻译**
   - 在"扫描文本"页面选择正确的路径
   - 工具会自动检测到 `.rpy` 文件并进行扫描

---

### 方案 B：通过 Ren'Py Modding API 创建翻译 Mod（绕过编译限制）

如果无法获取源代码，可以通过 Ren'Py Modding API 动态创建翻译模组：

**步骤：**

1. **创建 Mod 目录结构**
```
game/
└── mods/
    └── my_translation/
        ├── modname_manifest.json
        └── mod.rpy
```

2. **创建 manifest 文件** (`modname_manifest.json`)
```json
{
    "name": "my_translation",
    "version": "1.0.0",
    "load_order": "-inf",
    "init_label_name": "my_translation_init",
    "text_filter": "my_translation_text_filter",
    "text_messages": {
        "dialogue_1": "翻译的对话文本",
        "dialogue_2": "翻译的对话文本"
    }
}
```

3. **创建 mod.rpy**
```rpy
init -1 python:
    # 设置语言
    Game.language = "zh"

init 10 python:
    # 创建翻译字典
    translations = {
        "原始英文对话": "翻译成中文的对话",
        # ... 更多对话
    }
    
    def my_translation_text_filter(text):
        return translations.get(text, text)
    
    config.say_menu_text_filter = my_translation_text_filter

label my_translation_init:
    pass
```

**优点：**
- 不需要修改游戏源代码
- 完全兼容发行版本
- 支持通过 Mods 菜单启用/禁用

---

### 方案 C：使用 Ren'Py 的 .rpyc 反编译工具

某些第三方工具可以反编译 `.rpyc` 文件，但：

⚠️ **需要注意**
- 反编译可能不完全准确
- 可能违反游戏的使用条款
- 不推荐作为主要方案

如果确实需要，可搜索 `unrpa` 或 `rpyc decompiler`。

|------|---------|--------|--------|
| 有源代码版本 | `.rpy` | ✅ 完全支持 | ⭐⭐⭐⭐⭐ |

## 🆘 需要帮助？

1. **检查你的文件类型**
   - 在解压目录中查找文件
   - `.rpy` = 可以用 STS 工具翻译
   - `.rpyc` = 需要选择方案 B 或 C

2. **验证解压路径**
   - 应包含 `game/scripts/` 或类似的源代码结构
   - 里面应该有对话相关的 `.rpy` 文件（如 `dialogues.rpy`）

3. **联系开发者**
   - 如果来自官方的源代码版本仍然是 `.rpyc`
   - 请与游戏开发者确认是否有开发者友好的版本
