```bash
cd translator/
run_translator.bat      # Windows 双击
```

---

## ⚠️ 常见卡点 & 解决

### 卡点 1：扫描显示"未找到文件"

**原因**
```
检测到 .rpyc 编译文件（310 个）
.rpy 源代码文件（0 个）
```

**解决**
- ✅ 使用源代码版本（包含 `.rpy` 文件）
- ✨ 或通过 Modding API 创建翻译 Mod（无需源代码）
- 详见：[理解编译版本.md](理解编译版本.md)

---

### 卡点 2：API 连接失败

**检查清单**
- [ ] API Key 正确无误
- [ ] 网络连接正常
- [ ] 余额充足（付费 API）
- [ ] Base URL 格式正确
- [ ] 选择了有效的模型名称

**快速测试**
```bash
# 用 curl 测试 API
curl -X POST "https://api.openai.com/v1/chat/completions" \
  -H "Authorization: Bearer sk-YOUR-KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4o-mini","messages":[{"role":"user","content":"test"}]}'
```

---

### 卡点 3：翻译无反应

**可能原因**
- API 请求超时 → 降低"每批翻译数量"（默认 20）
- 网络中断 → 检查连接后重试
- API 繁忙 → 稍后重试或切换 API

**排查步骤**
1. 在"API 设置"点击 🔌 重新测试连接
2. 在"翻译"页面查看日志输出
3. 如仍失败，切换到本地 Ollama（完全免费）

## 🚀 推荐工作流

### 方案 A：完全免费（推荐新手）

```
1. 下载 Ollama（https://ollama.ai）
2. ollama pull qwen2.5
3. ollama serve
4. 工具设置：
   - Base URL: http://localhost:11434/v1
   - Model: qwen2.5
5. 开始翻译（无费用，无网络依赖）
```

### 方案 B：付费 API（更好质量）

```
1. 注册 OpenAI / DeepSeek
2. 获取 API Key
3. 充值余额
4. 工具设置对应的 Key 和 URL
5. 开始翻译
```

### 方案 C：无源代码的 Mod 方案

```
1. 不需要解压
2. 直接修改脚本目录为 game/scripts/
3. 扫描时会检测到 .rpyc，建议 Modding API
4. 参考"理解编译版本.md"的方案 B
```

## 🎮 集成翻译后的注意事项

### ✅ 正确做法
```
复制完整的 translation/ 文件夹到 game/scripts/
game/
└── scripts/
    └── translation/      ← 整个文件夹都要复制
        ├── set_language.rpy
        ├── text_filter.rpy
        └── dialogues_zh.rpy
```

### ❌ 错误做法
```
把翻译文件放到其他位置
或只放部分文件
```

## 🔧 手动编辑翻译

你可以用任何文本编辑器打开生成的 `.rpy` 文件进行修改：

```rpy
# 找到这样的对话
label bedroom_talk_zh:
    amy "我很高兴见到你。"  # ← 修改这里

# 改成你想要的内容
label bedroom_talk_zh:
    amy "哈喽！很高兴见到你呀。"
```

**修改后**
- 直接重启游戏即可加载新翻译
- 无需重新运行工具
